Prefix Sum

⏱ 5 min revision🎯 O(1) range queries📌 Frequency: Very High

1 Concept Overview

A prefix sum (cumulative sum) array lets you answer any subarray sum query in O(1) after O(n) preprocessing. Instead of summing elements from scratch for every query, you precompute a running total.

prefix[i] = sum of all elements from index 0 to i-1. Then the sum of any subarray arr[l..r] = prefix[r+1] - prefix[l].

IntuitionImagine measuring distances on a road. If you know the distance from city A to each city, you can compute the distance between any two cities instantly by subtraction — without redriving the route.

2 Why Interviewers Ask It

Prefix sum is the simplest example of "precomputation for faster queries." It tests whether you default to the O(n) naive approach or recognize the O(1) opportunity. It's also a prerequisite for sliding window, DP, and 2D grid optimizations.

3 Pattern Recognition

Use Prefix Sum when you see"Sum of subarray [l..r]" · "Range sum queries" · "Number of subarrays with sum k" · "Subarray equal to target" · "Running total" · Multiple sum queries on the same array

Key tell: if you're summing the same subarray multiple times, or checking many different subarrays for a sum condition, prefix sum is likely the trick.

4 Core Idea

Build the prefix array once. For 1-indexed prefix (most common interview style):

  • prefix[0] = 0
  • prefix[i] = prefix[i-1] + arr[i-1]
  • Sum of arr[l..r] = prefix[r+1] - prefix[l]
Off-by-one mattersThere are two common conventions. Be consistent. The 1-indexed prefix (size n+1) is usually cleaner — prefix[0]=0 acts as a sentinel that eliminates special casing for subarrays starting at index 0.

7 Common Tricks

🗃️

Prefix Sum + Hash Map

Store prefix sums in a hash map to find subarrays with sum exactly k in O(n). Key insight: if prefix[j] - prefix[i] = k, then subarray [i,j] sums to k.

🔢

2D Prefix Sum

For grid queries: prefix[i][j] = sum of rectangle from (0,0) to (i,j). Query any sub-rectangle in O(1) using inclusion-exclusion.

🔄

Difference Array

For range update queries (add k to all elements in [l,r]), use a difference array. Single-point query after all updates in O(1).

⚡

Modular Prefix Sum

For "number of subarrays divisible by k": track prefix[i] % k. If two prefixes have the same remainder, the subarray between them is divisible by k.

8 Common Mistakes

Mistake 1 — Off-by-one indexingSum of arr[l..r] = prefix[r+1] - prefix[l], NOT prefix[r] - prefix[l-1]. Draw it out first time.
Mistake 2 — Forgetting the 0-sentinelprefix[0] = 0 allows subarrays starting at index 0 to work without special casing. Never skip it.
Mistake 3 — Using prefix sum on mutable arraysPrefix sum is for static arrays. If the array changes, you'd need to rebuild the prefix array (O(n)) or use a Segment Tree / Binary Indexed Tree (O(log n) updates).

9 Python 2 Template

Basic Prefix Sum

Python 2
def build_prefix(arr):
    n = len(arr)
    prefix = [0] * (n + 1)   # 1-indexed, prefix[0] = 0 sentinel
    for i in xrange(n):
        prefix[i + 1] = prefix[i] + arr[i]
    return prefix

def range_sum(prefix, l, r):
    """Returns sum of arr[l..r] inclusive, 0-indexed."""
    return prefix[r + 1] - prefix[l]

# Example
arr = [3, 1, 4, 1, 5]
prefix = build_prefix(arr)
print range_sum(prefix, 1, 3)   # 1+4+1 = 6
print range_sum(prefix, 0, 4)   # 3+1+4+1+5 = 14

Subarray Sum Equals K (Prefix + Hash Map)

Python 2
# LC 560 — Count subarrays summing to k
def subarraySum(nums, k):
    count  = 0
    prefix = 0
    seen   = {0: 1}   # prefix_sum → how many times seen

    for n in nums:
        prefix += n
        # if (prefix - k) was seen before, subarray [that_index+1 .. here] sums to k
        count += seen.get(prefix - k, 0)
        seen[prefix] = seen.get(prefix, 0) + 1

    return count

10 Python 2 Variations

2D Prefix Sum

Python 2
def build_2d_prefix(matrix):
    m, n = len(matrix), len(matrix[0])
    p = [[0] * (n + 1) for _ in xrange(m + 1)]
    for i in xrange(1, m + 1):
        for j in xrange(1, n + 1):
            p[i][j] = (matrix[i-1][j-1]
                       + p[i-1][j] + p[i][j-1]
                       - p[i-1][j-1])
    return p

def region_sum(p, r1, c1, r2, c2):
    """Sum of rectangle (r1,c1) to (r2,c2) inclusive, 0-indexed."""
    return (p[r2+1][c2+1] - p[r1][c2+1]
            - p[r2+1][c1] + p[r1][c1])

Difference Array (range updates)

Python 2
# Add k to all elements in arr[l..r] for many queries, then read
n = 10
diff = [0] * (n + 1)

def range_add(l, r, k):
    diff[l]     += k
    diff[r + 1] -= k

def build_result():
    result = [0] * n
    running = 0
    for i in xrange(n):
        running    += diff[i]
        result[i]   = running
    return result

13 Mental Model

Think of prefix sum as an odometerYour car odometer tracks total distance traveled. To find distance between two cities, subtract the reading at city A from the reading at city B. You never re-drive the road — you just subtract. prefix[r+1] is odometer at r, prefix[l] is odometer at l.

14 Revision Notes

⚡ 1-Minute Review

  • prefix[0] = 0 (sentinel), prefix[i] = prefix[i-1] + arr[i-1]
  • sum(l, r) = prefix[r+1] - prefix[l] — O(1) query
  • Build O(n), query O(1). Total for Q queries: O(n + Q)
  • Prefix + hash map → count subarrays with sum k in O(n)
  • 2D version uses inclusion-exclusion for rectangle queries
  • For range updates: use difference array instead
  • Static array only — for updates use BIT / Segment Tree
