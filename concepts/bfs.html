Breadth-First Search (BFS)

⏱ 7 min revision🎯 O(V+E) time📌 Frequency: Very High

1 Concept Overview

BFS explores a graph level by level, visiting all nodes at distance 1 before distance 2, distance 2 before distance 3, and so on. It uses a queue (FIFO) to track the frontier.

The key property: BFS always finds the shortest path (fewest edges) in an unweighted graph. Once you reach a node, you've found the minimum-distance route.

IntuitionImagine dropping a stone in a pond. Ripples spread outward in concentric circles — that's BFS. Every ripple is one level. You reach every point at the nearest possible time.

2 Why Interviewers Ask It

BFS is the building block for shortest-path reasoning. Interviewers use it to test: can you translate a real-world problem into a graph and traverse it correctly? It's also the backbone of level-order tree traversal, word ladder, and grid problems.

3 Pattern Recognition

Use BFS when you see "Shortest path" · "Minimum steps/moves" · "Level order" · "Nearest X" · "Minimum number of operations" · Grid problems where each step costs 1

If the problem asks for minimum distance and all edges have equal weight (or unit cost), BFS is almost always the answer.

4 Core Idea

Queue + Visited set. Start by enqueuing the source node. Each iteration: dequeue a node, process it, enqueue all unvisited neighbors, mark them visited. Repeat until the queue is empty.

Key ruleMark nodes as visited when you enqueue them, not when you dequeue. Otherwise you'll enqueue the same node multiple times and blow up your time complexity.

6 Complexity

MetricTimeSpace
Graph (adjacency list)O(V + E)O(V)
Grid (m × n)O(m × n)O(m × n)
Tree (level order)O(n)O(w) — max width
Space caveatIn a balanced binary tree, the last level has n/2 nodes — so BFS space is O(n) in the worst case, same as DFS. But for wide graphs BFS uses more memory than DFS.

7 Common Tricks

📏

Track Distance / Level

Process the entire queue level at a time. Use for _ in xrange(len(queue)) at each BFS step to know when you finish one level.

🗺️

4-Directional Grid BFS

Define dirs = [(0,1),(0,-1),(1,0),(-1,0)] and iterate. Check bounds and visited status before enqueuing.

🔁

Multi-Source BFS

Enqueue ALL source nodes at the start (e.g. all '1' cells). BFS spreads from all sources simultaneously — finds nearest for every cell.

🔄

Bidirectional BFS

Search from both source and target simultaneously. Reduces search space from O(b^d) to O(b^(d/2)) — huge win for word ladder type problems.

8 Common Mistakes

Mistake 1 — Marking visited on dequeue, not enqueueCauses the same node to be enqueued multiple times, turning O(V+E) into O(V²) or worse in dense graphs.
Mistake 2 — Forgetting bounds check in grid BFSAlways validate 0 <= r < rows and 0 <= c < cols before enqueuing a grid cell.
Mistake 3 — Using BFS for weighted shortest pathBFS only works when all edges have equal weight. For weighted graphs, use Dijkstra.
Mistake 4 — Modifying the grid vs. using a visited setModifying grid cells in-place (marking '0') is fine for problems that don't need the original grid. If you need it unchanged, use a separate visited set.

9 Python 2 Template

General Graph BFS

Python 2
from collections import deque

def bfs(graph, start):
    visited = set([start])
    queue   = deque([start])
    dist    = {start: 0}

    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)       # mark on ENQUEUE
                dist[neighbor] = dist[node] + 1
                queue.append(neighbor)

    return dist

# Example: adjacency list
graph = {0: [1, 2], 1: [0, 3], 2: [0, 3], 3: [1, 2]}
print bfs(graph, 0)   # {0:0, 1:1, 2:1, 3:2}

Level-Order BFS (process level by level)

Python 2
from collections import deque

def bfs_levels(graph, start):
    visited = set([start])
    queue   = deque([start])
    result  = []

    while queue:
        level = []
        for _ in xrange(len(queue)):   # process entire level at once
            node = queue.popleft()
            level.append(node)
            for neighbor in graph[node]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
        result.append(level)

    return result

Grid BFS (4-directional)

Python 2
from collections import deque

def grid_bfs(grid, start_r, start_c):
    rows, cols = len(grid), len(grid[0])
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]
    visited = set([(start_r, start_c)])
    queue   = deque([(start_r, start_c, 0)])   # (row, col, dist)

    while queue:
        r, c, d = queue.popleft()
        if r == rows-1 and c == cols-1:        # reached target
            return d
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols and (nr,nc) not in visited:
                if grid[nr][nc] != '#':         # not a wall
                    visited.add((nr, nc))
                    queue.append((nr, nc, d+1))

    return -1   # not reachable

10 Python 2 Variations

Multi-Source BFS

Python 2
# Enqueue ALL sources at distance 0
# Classic: "distance of every cell from nearest 1" (LC 542)
from collections import deque

def multi_source_bfs(grid):
    rows, cols = len(grid), len(grid[0])
    dist = [[float('inf')] * cols for _ in xrange(rows)]
    queue = deque()

    for r in xrange(rows):
        for c in xrange(cols):
            if grid[r][c] == 1:
                dist[r][c] = 0
                queue.append((r, c))

    dirs = [(0,1),(0,-1),(1,0),(-1,0)]
    while queue:
        r, c = queue.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols:
                if dist[nr][nc] > dist[r][c] + 1:
                    dist[nr][nc] = dist[r][c] + 1
                    queue.append((nr, nc))
    return dist

12 Step-by-Step Walkthrough

Problem: Shortest path from (0,0) to (2,2) in a 3×3 grid (no obstacles).

1

Initialize

queue = [(0,0,0)], visited = {(0,0)}. Start at top-left, distance 0.

2

Process (0,0)

Neighbors: (0,1) and (1,0). Enqueue both with dist=1. visited={(0,0),(0,1),(1,0)}.

3

Process (0,1)

New neighbors: (0,2) and (1,1). Enqueue with dist=2.

4

Process (1,0)

New neighbor: (2,0) and (1,1) already visited. Enqueue (2,0) with dist=2.

5

Continue until (2,2)

Eventually (2,2) is dequeued with dist=4. That's the answer: 4 steps.

13 Mental Model

Think of BFS as a spreading fireYou light a fire at the source. It spreads to all adjacent cells in 1 second, then from those cells in the next second. Every cell is reached at the earliest possible time. BFS guarantees you see the shortest path first.

14 Revision Notes

⚡ 1-Minute Review

  • Use a queue (deque). Never a stack for BFS.
  • Mark visited on enqueue, not dequeue
  • Guarantees shortest path in unweighted graphs only
  • Level-order: use for _ in xrange(len(queue)) to process one level
  • Grid BFS: 4 directions, bounds check, visited set or in-place mutation
  • Multi-source: enqueue ALL sources at dist 0 before starting the loop
  • For weighted graphs → Dijkstra, not BFS
