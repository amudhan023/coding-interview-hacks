Redundant Connection LC #684

⏱ 7 min revision🎯 Union Find Cycle Detection📌 Frequency: High

1 Problem Summary

ProblemYou are given a tree with n nodes numbered 1 to n, plus one extra edge that creates a cycle. Return that extra (redundant) edge. If multiple answers exist, return the last one in the input.

Example: edges = [[1,2],[1,3],[2,3]] → [2,3]

1–2 and 1–3 form a valid tree. Adding 2–3 creates a cycle: 1–2–3–1. The redundant edge is [2,3].

2 Pattern

Union Find Cycle DetectionProcess edges one by one. For each edge [u, v]: if find(u) == find(v), they're already connected — this edge creates a cycle → it's the redundant one. Otherwise, union(u, v) and continue.

Key insight: in a tree, every edge connects two different components. The first edge where both endpoints are already in the same component is the redundant edge.

3 Visualization

4 Brute Force — DFS O(n²)

For each edge, check if a path already exists between its endpoints using DFS. If yes, this edge is redundant. O(n) per check × O(n) edges = O(n²).

Python 2 — Brute Force
class Solution(object):
    def findRedundantConnection(self, edges):
        graph = {}

        def has_path(src, dst, visited):
            if src == dst: return True
            visited.add(src)
            for nb in graph.get(src, []):
                if nb not in visited and has_path(nb, dst, visited):
                    return True
            return False

        for u, v in edges:
            if has_path(u, v, set()):   # path already exists?
                return [u, v]
            graph.setdefault(u, []).append(v)
            graph.setdefault(v, []).append(u)

        return []

5 Optimal Solution — Union Find O(n · α(n))

Union Find tells you in near-O(1) whether two nodes are already connected. Process each edge: if both endpoints have the same root, you found the redundant edge. Otherwise, union them.

The elegant ruleA redundant edge is one where both endpoints are already in the same connected component when we try to process it. Union Find detects this instantly.

6 Python 2 Code

Python 2 — Union Find (Optimal)
class Solution(object):
    def findRedundantConnection(self, edges):
        n = len(edges)
        parent = range(n + 1)   # 1-indexed nodes
        rank   = [0] * (n + 1)

        def find(x):
            if parent[x] != x:
                parent[x] = find(parent[x])   # path compression
            return parent[x]

        def union(x, y):
            px, py = find(x), find(y)
            if px == py:
                return False   # already connected — redundant edge
            if rank[px] < rank[py]:
                px, py = py, px
            parent[py] = px
            if rank[px] == rank[py]:
                rank[px] += 1
            return True

        for u, v in edges:
            if not union(u, v):
                return [u, v]

        return []

find(1)=1, find(4)=1. Same root! Union returns False. Return [1,4].

8 Complexity Analysis

ApproachTimeSpace
Brute force DFS per edgeO(n²)O(n)
Union Find (with path compression + rank)O(n · α(n)) ≈ O(n)O(n)
Why Union Find wins hereDFS checks full connectivity on each new edge — O(n) per edge. Union Find maintains connectivity incrementally — O(α(n)) per edge, effectively constant.

9 Variations

🔢

Redundant Connection II (LC 685)

Directed graph. Harder — handle two cases: node with two parents, or cycle without such a node. Requires careful Union Find with rollback.

🌳

Kruskal's MST

Same Union Find logic — process edges sorted by weight, skip edges where both endpoints share a root. The accepted edges form the MST.

🔗

Graph Valid Tree (LC 261)

n nodes and n-1 edges? Check: 1) No cycle (Union Find), 2) All connected (count=1). Both conditions → valid tree.

🌐

Number of Connected Components

Same Union Find structure. Count how many nodes have parent[x]==x after all unions. That's the component count.
