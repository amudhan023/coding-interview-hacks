Dijkstra's Shortest Path

⏱ 10 min revision🎯 O((V+E) log V)📌 Frequency: High

1 Concept Overview

Dijkstra's algorithm finds the shortest path from a source node to all other nodes in a weighted graph with non-negative edge weights. It uses a min-heap (priority queue) to always expand the cheapest known node first.

Think of it as BFS but with a cost-aware frontier: instead of a simple queue, you use a min-heap keyed on cost-so-far.

Critical constraintDijkstra requires all edge weights to be non-negative. For negative weights, use Bellman-Ford instead.

2 Why Interviewers Ask It

Dijkstra tests your understanding of greedy algorithms and heap usage. It's a harder version of BFS that comes up in network routing, GPS pathfinding, and game AI. The heap management and relaxation logic reveal whether you truly understand priority-queue-based algorithms.

3 Pattern Recognition

Use Dijkstra when you see"Shortest path" in a weighted graph · "Minimum cost path" · "Cheapest route" · "Network delay time" · Any graph with non-negative edge costs where you need minimum total cost

Key tell: the graph has weighted edges AND you need the minimum total weight path. If weights are all 1 (or equal), use BFS — it's faster.

4 Core Idea

Greedy expansion via min-heap. Maintain a distance table dist[] initialized to infinity for all nodes except the source (dist[src] = 0). Use a min-heap to always expand the node with the current minimum known distance.

When you expand node u with cost d, for each neighbor v with edge weight w: if d + w < dist[v], update dist[v] = d + w and push (d+w, v) to the heap. This is called relaxation.

Why greedy works hereOnce you pop a node from the min-heap, you have found its shortest path — no future path can be shorter (because all weights are non-negative). So you can skip it if you've seen it before.

6 Complexity

ImplementationTimeSpace
Binary heap + adjacency listO((V + E) log V)O(V + E)
Fibonacci heap (theoretical)O(E + V log V)O(V)
Simple array (dense graphs)O(V²)O(V)

In interviews, always use binary heap (Python's heapq). Never mention Fibonacci heap unless specifically asked.

7 Common Tricks

⏭️

Skip Stale Entries

Python's heapq doesn't support decrease-key. Instead, push duplicates and skip when d > dist[node] on pop. Simpler and correct.

🎯

Early Termination

If you only need the shortest path to a specific target, return as soon as you pop that target from the heap.

🔄

Reconstruct Path

Keep a prev[] dictionary: prev[v] = u when you relax edge (u,v). Trace back from target to source to get the full path.

🌐

K Shortest Paths

Allow each node to be popped up to K times. The K-th pop gives the K-th shortest path (Yen's algorithm variant).

8 Common Mistakes

Mistake 1 — Using Dijkstra with negative weightsDijkstra's greedy assumption breaks with negative edges. Use Bellman-Ford (O(VE)) for negative weights.
Mistake 2 — Not skipping stale heap entriesWithout if d > dist[node]: continue, you reprocess nodes with outdated distances, ruining correctness and performance.
Mistake 3 — Pushing to heap without checkingAlways check if the new distance is actually better before pushing: if dist[node] + w < dist[nb]: push.
Mistake 4 — Initializing dist as {}Initialize ALL nodes to infinity upfront — dist = [float('inf')] * n. Missing nodes will cause KeyErrors mid-traversal.

9 Python 2 Template

Optimal — heapq with stale-skip

Python 2
import heapq

def dijkstra(graph, src, n):
    """
    graph: adjacency list — graph[u] = [(v, weight), ...]
    src:   source node
    n:     number of nodes
    Returns: dist[] — shortest distances from src to all nodes
    """
    dist = [float('inf')] * n
    dist[src] = 0
    heap = [(0, src)]   # (cost, node)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]:        # stale entry — skip
            continue
        for v, w in graph[u]:
            new_d = d + w
            if new_d < dist[v]:
                dist[v] = new_d
                heapq.heappush(heap, (new_d, v))

    return dist


# Example: graph as adjacency list
graph = [[] for _ in xrange(5)]
graph[0].append((1, 4))
graph[0].append((2, 2))
graph[2].append((3, 1))
graph[1].append((4, 3))
graph[3].append((4, 1))

print dijkstra(graph, 0, 5)   # [0, 4, 2, 3, 4]

With Path Reconstruction

Python 2
import heapq

def dijkstra_path(graph, src, dst, n):
    dist = [float('inf')] * n
    prev = [-1] * n
    dist[src] = 0
    heap = [(0, src)]

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue
        if u == dst: break         # early exit if we only need one target
        for v, w in graph[u]:
            if d + w < dist[v]:
                dist[v] = d + w
                prev[v] = u
                heapq.heappush(heap, (dist[v], v))

    # Reconstruct path
    path = []
    node = dst
    while node != -1:
        path.append(node)
        node = prev[node]
    return dist[dst], path[::-1]

10 Python 2 Variations

Dijkstra on grid (variable movement cost)

Python 2
import heapq

def grid_dijkstra(grid):
    rows, cols = len(grid), len(grid[0])
    dist = [[float('inf')] * cols for _ in xrange(rows)]
    dist[0][0] = grid[0][0]
    heap = [(grid[0][0], 0, 0)]

    dirs = [(0,1),(0,-1),(1,0),(-1,0)]
    while heap:
        d, r, c = heapq.heappop(heap)
        if d > dist[r][c]: continue
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols:
                new_d = d + grid[nr][nc]
                if new_d < dist[nr][nc]:
                    dist[nr][nc] = new_d
                    heapq.heappush(heap, (new_d, nr, nc))

    return dist[rows-1][cols-1]

12 Step-by-Step Walkthrough

Graph: A→B(4), A→C(2), C→D(1), B→E(3), D→E(1). Find shortest path from A.

1

Init

dist=[0,∞,∞,∞,∞], heap=[(0,A)]

2

Pop (0, A)

Relax B: dist[B]=4, push (4,B). Relax C: dist[C]=2, push (2,C). heap=[(2,C),(4,B)]

3

Pop (2, C)

Relax D: 2+1=3 < ∞, dist[D]=3, push (3,D). heap=[(3,D),(4,B)]

4

Pop (3, D)

Relax E: 3+1=4 < ∞, dist[E]=4, push (4,E). heap=[(4,B),(4,E)]

5

Pop (4, B)

Relax E: 4+3=7 > dist[E]=4. No update. Pop (4,E) — already optimal. Done.

6

Result

dist = [0, 4, 2, 3, 4]. Shortest path A→E = 4 via A→C→D→E.

13 Mental Model

Think of Dijkstra as "spreading ink through pipes"You pour ink at the source. It flows through pipes — wider pipes (lower weight) let it flow faster. The ink reaches each city at the exact minimum time possible. Once the ink reaches a city, that time is final and optimal. Dijkstra is just tracking when the ink arrives at each city.

14 Revision Notes

⚡ 1-Minute Review

  • Min-heap keyed on (cost, node)
  • Init: dist[src]=0, all others=∞
  • On pop: skip if d > dist[node] (stale entry)
  • Relax: if d + w < dist[v], update and push
  • Non-negative weights only — use Bellman-Ford for negatives
  • Time: O((V+E) log V) with binary heap
  • Early termination: return when target is popped
  • Path reconstruction: keep prev[] array
