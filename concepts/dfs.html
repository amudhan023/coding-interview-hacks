Depth-First Search (DFS)

⏱ 7 min revision🎯 O(V+E) time📌 Frequency: Very High

1 Concept Overview

DFS explores a graph by going as deep as possible along each branch before backtracking. It uses a stack — either the call stack (recursion) or an explicit stack.

Unlike BFS, DFS doesn't guarantee shortest path. Its strength is in exploring all possibilities: cycle detection, topological sort, connected components, and combinatorial search (backtracking).

IntuitionImagine exploring a maze. DFS picks a direction and goes all the way to the end (or a dead end), then backtracks and tries the next direction. It's "go deep, then backtrack" vs BFS's "go wide."

2 Why Interviewers Ask It

DFS tests recursion fluency, cycle detection, and backtracking — skills that matter for tree problems, graph problems, and combinatorial search. It's also the basis for topological sort (Kahn's or recursive DFS), which appears heavily in dependency/scheduling problems.

3 Pattern Recognition

Use DFS when you see"Explore all paths" · "All combinations/permutations" · "Cycle detection" · "Topological order" · "Is there a path?" · "Tree traversals (in/pre/post-order)" · Backtracking problems

BFS finds the shortest path. DFS finds any path, or checks all paths. If the problem doesn't require minimum distance, DFS is usually simpler to code.

4 Core Idea

Recurse into neighbors, mark visited, backtrack. The recursion call stack automatically acts as your stack. For very deep graphs (risk of recursion limit), use an explicit stack with iteration.

Python recursion limitPython's default recursion limit is 1000. For deep graphs, add sys.setrecursionlimit(10000) or convert to iterative DFS.

7 Common Tricks

🔄

Three Colors for Cycle Detection

WHITE=0 (unvisited), GRAY=1 (in current path), BLACK=2 (done). If you hit GRAY, you found a back edge (cycle).

📋

Topological Sort

Do DFS, append node to result AFTER all its children are processed (post-order). Reverse the result.

↩️

Backtracking Pattern

Before recursing: add to state. After recursing: remove from state. This "undo" is the heart of backtracking.

🔢

Iterative DFS

Replace recursion with an explicit stack. Push neighbors in reverse order to maintain left-to-right traversal order.

8 Common Mistakes

Mistake 1 — Not tracking visited nodes in graphs (vs. trees)Trees have no cycles, so visited tracking is optional. Graphs can have cycles — without visited, DFS loops forever.
Mistake 2 — Forgetting to undo state in backtrackingIf you append to a path before recursing, you must pop after. Missing the undo step corrupts the path for sibling branches.
Mistake 3 — Using DFS for shortest pathDFS finds a path, not the shortest path. Use BFS for unweighted shortest paths.
Mistake 4 — Stack overflow on large inputsPython's recursion limit is 1000. Use iterative DFS or increase the limit with sys.setrecursionlimit() for large graphs.

9 Python 2 Template

Recursive DFS — Graph

Python 2
def dfs(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
    return visited

# Example
graph = {0:[1,2], 1:[0,3], 2:[0], 3:[1]}
print dfs(graph, 0)   # {0, 1, 2, 3}

Iterative DFS — Graph

Python 2
def dfs_iterative(graph, start):
    visited = set()
    stack   = [start]
    order   = []

    while stack:
        node = stack.pop()
        if node in visited:
            continue
        visited.add(node)
        order.append(node)
        for neighbor in reversed(graph[node]):   # reversed to preserve order
            if neighbor not in visited:
                stack.append(neighbor)

    return order

Cycle Detection (directed graph — 3 colors)

Python 2
WHITE, GRAY, BLACK = 0, 1, 2

def has_cycle(graph, n):
    color = [WHITE] * n

    def dfs(node):
        color[node] = GRAY
        for nb in graph[node]:
            if color[nb] == GRAY:   # back edge = cycle
                return True
            if color[nb] == WHITE and dfs(nb):
                return True
        color[node] = BLACK
        return False

    return any(dfs(i) for i in xrange(n) if color[i] == WHITE)

Backtracking Template

Python 2
def backtrack(candidates, target, start, path, result):
    if target == 0:
        result.append(list(path))   # found a valid combination
        return
    for i in xrange(start, len(candidates)):
        if candidates[i] > target:
            break
        path.append(candidates[i])          # choose
        backtrack(candidates, target - candidates[i], i, path, result)
        path.pop()                          # undo (backtrack)

10 Python 2 Variations

Topological Sort (DFS post-order)

Python 2
def topological_sort(graph, n):
    visited = set()
    result  = []

    def dfs(node):
        visited.add(node)
        for nb in graph[node]:
            if nb not in visited:
                dfs(nb)
        result.append(node)   # post-order: add AFTER children

    for node in xrange(n):
        if node not in visited:
            dfs(node)

    return result[::-1]   # reverse gives topological order

13 Mental Model

Think of DFS as "commit fully, then reconsider"You're navigating a choose-your-own-adventure book. DFS reads every path to its end before flipping back to try a different choice. BFS reads all options at chapter 1, then all options at chapter 2. DFS dives all the way to the final chapter first.

14 Revision Notes

⚡ 1-Minute Review

  • Uses a stack (recursion call stack, or explicit)
  • Does NOT guarantee shortest path — use BFS for that
  • Tree DFS: no visited set needed (no cycles in trees)
  • Graph DFS: always track visited to avoid infinite loops
  • Cycle detection: use WHITE/GRAY/BLACK coloring for directed graphs
  • Backtracking = DFS + undo state after each recursive call
  • Topological sort = DFS post-order, then reverse
  • Python recursion limit = 1000. Use iterative for deep graphs.
