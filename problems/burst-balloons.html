Burst Balloons LC #312

⏱ 12 min revision🎯 Interval DP — "Last to Burst"📌 Frequency: Medium-High

1 Problem Summary

ProblemGiven n balloons with values nums[0..n-1], if you burst balloon i you earn nums[i-1] × nums[i] × nums[i+1] coins. After bursting, the neighbors become adjacent. Burst all balloons to maximize total coins.

Example: nums = [3, 1, 5, 8] → Output: 167

The trapThe naive approach is "which balloon should I burst first?" — but every burst changes the neighbors. After bursting balloon i, the boundary is now (i-1) and (i+1) which are a new pair. Tracking all possible orderings = O(n!) × O(n) per ordering = completely infeasible.

2 Pattern Recognition

Interval DP — Think about the LAST balloon to burstInstead of "which burst first?" (unpredictable neighbors), ask "which balloon bursts LAST in range [l..r]?" The last balloon's neighbors are always nums[l-1] and nums[r+1] — the boundaries of the range, which are guaranteed to still be present.

3 The "Last Balloon" Reversal — The Aha Moment

This is one of the cleanest example of interval DP thinking. The key reversal:

❌

Wrong Question

"Which balloon should I burst first?" — When balloon k bursts, its coins depend on its current neighbors, which have already changed from earlier bursts. Unpredictable.

✅

Right Question

"Which balloon bursts last in range [l..r]?" — When k is last, all others in [l..r] are already gone. So k's neighbors are exactly nums[l-1] and nums[r+1]. Fully determined!

The Recurrence
Pad array: nums = [1] + nums + [1] (virtual balloons worth 1 at both ends)

dp[l][r] = max coins from bursting all balloons in range [l..r]

For each possible "last balloon" k in [l..r]:
dp[l][r] = max( dp[l][k-1] + nums[l-1]×nums[k]×nums[r+1] + dp[k+1][r] )

Fill by increasing range length: length 1 → 2 → 3 → ... → n
Answer = dp[1][n]

5 Brute Force — O(n! × n)

Try every permutation of balloon burst order. For each permutation, simulate the bursts and track coins. This is completely infeasible for n=30+.

Python 2 — Brute Force (TLE)
from itertools import permutations

def maxCoins_brute(nums):
    best = 0
    for perm in permutations(range(len(nums))):
        arr = list(nums)
        coins = 0
        for i in perm:
            # Rebuild current neighbors
            left  = arr[i-1] if i > 0 else 1
            right = arr[i+1] if i < len(arr)-1 else 1
            # ... this approach fails because indices shift after deletions
        best = max(best, coins)
    return best   # Incorrect + O(n! * n)
Why brute force failsAfter each burst, indices shift. Tracking the changing array state while iterating over permutations is both complex and O(n!) — completely unusable.

6 Optimal Solution — Interval DP O(n³)

1

Pad the array

Add virtual 1s: nums = [1] + nums + [1]. This eliminates out-of-bounds checking. Work on indices 1..n.

2

Define dp[l][r]

Maximum coins from bursting all balloons strictly inside the boundary [l-1, r+1]. The l-1 and r+1 balloons are the "walls" that are NOT burst in this subproblem.

3

Fill by increasing length

For length 1 (each single balloon), then length 2, ..., up to length n. Each subproblem depends only on smaller subproblems already computed.

4

For each [l,r], try every k as last

dp[l][r] = max over k∈[l,r]: dp[l][k-1] + nums[l-1]×nums[k]×nums[r+1] + dp[k+1][r]

5

Answer

dp[1][n] is the max coins for the entire array.

7 Python 2 Code

Python 2 — Interval DP (Optimal)
class Solution(object):
    def maxCoins(self, nums):
        # Step 1: Pad with virtual 1s at both ends
        nums = [1] + nums + [1]
        n    = len(nums)

        # dp[l][r] = max coins bursting all balloons in (l-1, r+1) exclusive boundaries
        # i.e., balloons at indices l through r (using 1-indexed real balloons)
        dp = [[0] * n for _ in xrange(n)]

        # Step 2: Fill by increasing range length
        for length in xrange(1, n - 1):       # length of range [l..r]
            for l in xrange(1, n - length):    # left boundary
                r = l + length - 1             # right boundary

                # Step 3: Try every k as the LAST balloon to burst in [l..r]
                for k in xrange(l, r + 1):
                    coins = (nums[l-1] * nums[k] * nums[r+1]   # burst k last
                             + dp[l][k-1]                       # left subproblem
                             + dp[k+1][r])                      # right subproblem
                    dp[l][r] = max(dp[l][r], coins)

        # Answer: burst all real balloons (indices 1 to n-2)
        return dp[1][n - 2]


# Example
sol = Solution()
print sol.maxCoins([3, 1, 5, 8])   # 167
print sol.maxCoins([1, 5])          # 10

Top-down Memoization (Alternative)

Python 2 — Top-down with Memo
class Solution(object):
    def maxCoins(self, nums):
        nums = [1] + nums + [1]
        memo = {}

        def dp(l, r):
            if l > r: return 0
            if (l, r) in memo: return memo[(l, r)]
            best = 0
            for k in xrange(l, r + 1):
                coins = (nums[l-1] * nums[k] * nums[r+1]
                         + dp(l, k-1) + dp(k+1, r))
                best = max(best, coins)
            memo[(l, r)] = best
            return best

        return dp(1, len(nums) - 2)

8 Complete Dry Run — nums = [3, 1, 5, 8]

Padded: [1, 3, 1, 5, 8, 1] → indices 0..5. Real balloons at 1..4.

Length-1 subproblems (single balloon, boundaries are immediate neighbors in padded array)
dp[1][1]: k=1 → 1×3×1 = 3
dp[2][2]: k=2 → 3×1×5 = 15
dp[3][3]: k=3 → 1×5×8 = 40

9 Complexity Analysis

ApproachTimeSpace
Brute force (all orderings)O(n! × n)O(n)
Interval DP (bottom-up)O(n³)O(n²)
Interval DP (top-down memo)O(n³)O(n²)

n³ because: O(n²) subproblems × O(n) choices of k per subproblem. For n=500, that's 125M operations — fast enough.

10 Common Mistakes

Mistake 1 — Forgetting to padWithout the 1-padding, you need special boundary checks everywhere. Always add [1] + nums + [1] first.
Mistake 2 — Wrong range length loopThe outer loop must iterate by length (not just l,r independently) to ensure smaller subproblems are filled before larger ones.
Mistake 3 — Off-by-one in boundariesnums[l-1] and nums[r+1] are the boundary values (outside the range). k iterates from l to r inclusive.
Mistake 4 — Thinking of "first" instead of "last"dp[l][k-1] and dp[k+1][r] are the subproblems for everything OTHER than k. They're solved assuming k hasn't burst yet (because k is last). If you reverse the logic, the coins formula breaks.

11 Revision Notes

⚡ 1-Minute Review

  • Think "which balloon bursts last in [l,r]?" — not first
  • When k is last: its neighbors are fixed at nums[l-1] and nums[r+1]
  • Pad with 1s: nums = [1] + nums + [1]
  • Recurrence: dp[l][r] = max(dp[l][k-1] + nums[l-1]*nums[k]*nums[r+1] + dp[k+1][r])
  • Fill by increasing length (len=1 → 2 → ... → n)
  • O(n³) time, O(n²) space
  • Answer = dp[1][n]
