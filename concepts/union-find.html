Union Find (Disjoint Set Union)

⏱ 8 min revision 🎯 O(α(n)) per operation 📌 Frequency: Very High

1 Concept Overview

Union Find (also called Disjoint Set Union or DSU) is a data structure that tracks which elements belong to the same connected component. Think of it as a forest of trees where each tree represents one group.

You have two core operations:

  • find(x) — Which group does x belong to? Returns the root of x's tree.
  • union(x, y) — Merge the groups of x and y into one.
Intuition Imagine students in a classroom. Initially everyone is their own group. When two students become friends, their friend groups merge. find tells you which friend group someone belongs to. union merges two friend groups.

2 Why Interviewers Ask It

Union Find tests whether you can recognize dynamic connectivity problems — problems where groups form incrementally. It's a clean separator between candidates who know data structures deeply and those who only know BFS/DFS.

It appears in: Kruskal's MST, cycle detection in undirected graphs, grouping problems, and any "how many connected components" question where edges arrive one at a time.

3 Pattern Recognition

Use Union Find when you see "Connected components" · "Are X and Y in the same group?" · "How many distinct groups?" · "Does adding this edge create a cycle?" · "Merge groups dynamically"

Key tell: the problem adds edges/relationships one at a time and asks whether two nodes are connected. BFS/DFS rebuild everything from scratch — Union Find is incremental and faster.

4 Core Idea

Each element starts as its own parent (parent[i] = i). To find the root, follow parent pointers until you reach a node that points to itself. To union two elements, set one's root to be the parent of the other.

Two critical optimizations make it nearly O(1):

🔗

Path Compression

During find, make every node on the path point directly to the root. Flattens the tree for future lookups.

⚖️

Union by Rank

Always attach the smaller tree under the root of the larger tree. Keeps trees shallow. Use rank[] or size[].

6 Complexity

OperationTimeSpace
find (with path compression)O(α(n)) ≈ O(1)—
union (with rank)O(α(n)) ≈ O(1)—
InitializationO(n)O(n)
m operations on n elementsO(m · α(n))O(n)
α(n) — Inverse Ackermann Function For all practical purposes α(n) ≤ 4. The function grows so slowly that it's effectively constant. Never say O(1) in interviews — say "inverse Ackermann, essentially constant."

7 Common Tricks

⚡

Recursive Path Compression

One-liner: parent[x] = find(parent[x]). Clean and effective. Points every ancestor directly to root during the find call itself.

📊

Track Component Count

Initialize count = n. Decrement by 1 inside union only when the roots differ. Gives you live component count for free.

🎯

Cycle Detection

If find(u) == find(v) before calling union(u,v), the edge (u,v) creates a cycle — u and v are already connected.

🔢

Union by Size vs Rank

Use size[] instead of rank[] when you need to know component sizes. Merge smaller under larger. Same asymptotic performance.

8 Common Mistakes

Mistake 1 — Forgetting path compression Without it, find() is O(n) in the worst case. Always use parent[x] = find(parent[x]).
Mistake 2 — Union without checking if roots are the same Calling union on already-connected nodes corrupts rank. Always check find(x) != find(y) before merging.
Mistake 3 — Forgetting to decrement component count If the problem asks for the number of components, you need to track this explicitly.
Mistake 4 — Using Union Find for directed graphs Union Find works on undirected graphs only. For directed graphs use DFS/topological sort.

9 Python 2 Template

Optimal (Path Compression + Union by Rank)

Python 2 
class UnionFind(object):
    def __init__(self, n):
        self.parent = range(n)        # parent[i] = i initially
        self.rank   = [0] * n
        self.count  = n               # number of components

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return False              # already connected, would create cycle
        # union by rank
        if self.rank[px] < self.rank[py]:
            px, py = py, px           # ensure px has higher rank
        self.parent[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1
        self.count -= 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)


# Example usage
uf = UnionFind(5)
uf.union(0, 1)
uf.union(2, 3)
print uf.connected(0, 1)   # True
print uf.connected(0, 2)   # False
print uf.count             # 3 components: {0,1}, {2,3}, {4}

Minimal (Interviews — fits in 10 lines)

Python 2 
parent = range(n)

def find(x):
    while parent[x] != x:
        parent[x] = parent[parent[x]]   # path compression (halving)
        x = parent[x]
    return x

def union(x, y):
    px, py = find(x), find(y)
    if px != py:
        parent[px] = py

10 Python 2 Variations

Union by Size (track component sizes)

Python 2 
class UnionFind(object):
    def __init__(self, n):
        self.parent = range(n)
        self.size   = [1] * n   # size of each component
        self.count  = n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False
        if self.size[px] < self.size[py]:
            px, py = py, px
        self.parent[py] = px
        self.size[px]  += self.size[py]
        self.count     -= 1
        return True

    def get_size(self, x):
        return self.size[self.find(x)]

parent = range(2 * n)

def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])
    return parent[x]

def union_enemies(x, y):
    # x and y are enemies: connect x with y's enemy, y with x's enemy
    union(x, y + n)
    union(x + n, y)

12 Step-by-Step Walkthrough

Problem: Given edges [[1,2],[2,3],[3,4],[1,4],[1,5]], find the redundant connection.

1

Initialize

parent = [0,1,2,3,4,5], count = 5. Each node is its own component.

2

union(1, 2)

find(1)=1, find(2)=2. Different roots → merge. parent[2]=1. count=4.

3

union(2, 3)

find(2)=1, find(3)=3. Different → merge. parent[3]=1. count=3.

4

union(3, 4)

find(3)=1, find(4)=4. Different → merge. parent[4]=1. count=2.

5

union(1, 4) — Cycle!

find(1)=1, find(4)=1. Same root! This edge [1,4] would create a cycle. Return it as the redundant connection.

13 Mental Model

Think of it as a "club membership" system Each person starts in their own solo club. union(x, y) = merge club x and club y. find(x) = who is the president of x's club? Two people are in the same club iff they have the same president. Path compression = every member updates their records to know the president directly (skip the chain).

14 Revision Notes

⚡ 1-Minute Review

  • Two operations: find(x) → root of x's tree; union(x,y) → merge trees
  • Path compression: parent[x] = find(parent[x]) — flattens tree on the fly
  • Union by rank: attach smaller tree under larger — keeps height O(log n)
  • Together → O(α(n)) ≈ O(1) per operation
  • Cycle detection: if find(u) == find(v), edge (u,v) creates a cycle
  • Track component count: start at n, decrement each successful union
  • Works on undirected graphs only
