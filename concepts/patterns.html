Top Interview 150 Pattern Analysis

Hidden Patterns Across the Top 150

⏱ 15 min read 🎯 20 ranked patterns 📌 Ranked by interview frequency

These aren't category labels. They're reusable thinking instincts — the mental shortcuts that let experienced engineers solve problems in minutes, not hours. Each pattern is ranked by how many Top 150 problems it unlocks and how much time it saves under interview pressure.

TIER 1

Must Know — Core Patterns

Appears in 10+ problems each
#1
Hash Map as O(1) State
The universal lookup accelerator — covers ~30 problems in the Top 150
🔥 Highest Frequency Two Sum Complement Frequency Counter Visited Set Memoization

Store things you've already seen so you can answer "have I seen this before?" or "what's the complement of this?" in O(1) instead of O(n). The map stores the answer to your question, not just the value.

🎯 Trigger

"Does X exist in the array?" · "Count of each element" · "Complement/partner of X" · "Group elements by key" · "Have I seen this state before?"

📌 Example — Two Sum

For each num, check if target - num is in the map. Store {value: index} as you scan. O(n) instead of O(n²).

❌ Common Mistake

Using a list/set when you need the index too. Or forgetting to update the map before checking — order matters for self-pair avoidance.

⏱ Saves

Avoids the O(n²) brute force. Valid Anagram, Group Anagrams, Ransom Note, Isomorphic Strings, Word Pattern, Happy Number, Longest Consecutive all reduce to this.

Mental shortcutWhen you think "I need to search for something I've seen before," reach for a dict. The question to ask: "What is the key, and what is the value I need to store?"
#2
Two Pointers — Inward Squeeze & Gap Pointer
Two flavors: (L,R) squeezing from both ends, or (slow, fast) with a gap
🔥 Very High Frequency Sorted Array Pair Remove Nth from End Palindrome Check

Inward squeeze: left and right pointers move toward each other — used when the answer improves as pointers converge on a sorted (or near-sorted) structure. Gap pointer: two pointers separated by k steps — used for "kth from end" or "intersection" problems.

🎯 Trigger

"Find pair summing to target" in sorted array · "Maximize area/sum" · "Is this a palindrome?" · "Remove nth from end" · "Is A a subsequence of B?"

📌 Example — 3Sum

Sort, fix one element, then squeeze two pointers on the rest. Skip duplicates by checking if nums[i] == nums[i-1]: continue at each level.

❌ Common Mistake

Forgetting to advance both pointers when a match is found in 3Sum, leading to infinite loops. Also: using two pointers on an unsorted array when they depend on order.

⏱ Saves

Turns O(n²) brute force into O(n). Container With Most Water becomes one-pass. Remove Nth Node becomes one-pass with a dummy head.

#3
Tree Recursion Return Value
The recursive function returns richer information than just the answer
🔥 Covers 12+ Tree Problems Post-order Subtree Info Global Variable

Most binary tree problems that feel different are the same recursion template. The function returns what the parent needs to know about this subtree. A global variable captures the overall answer. The key question is: "What does the parent need from me?"

🎯 Trigger

Any problem that asks you to aggregate info across a tree: max path, diameter, balance, LCA, max/min depth, count good nodes.

📌 Example — Diameter

Return height to parent. At each node, update self.ans = max(self.ans, left_h + right_h). The diameter is the max across all such sums.

❌ Common Mistake

Trying to compute the answer in the return value instead of a nonlocal/global. The return value is for the parent; the global accumulates the final answer.

⏱ Saves

One template handles: Max Depth, Diameter, Balanced Tree, Max Path Sum, LCA, Count Good Nodes, Binary Tree Cameras. Recognize the shape, swap the return value.

Python 2 — Universal Tree Template
self.ans = 0   # or -inf, or float('inf')

def dfs(node):
    if not node: return BASE_VALUE   # e.g. 0 for height, float('-inf') for max path
    left  = dfs(node.left)
    right = dfs(node.right)
    # Update global answer using left + right info
    self.ans = max(self.ans, COMBINE(left, right, node.val))
    # Return what the PARENT needs from this subtree
    return RETURN_TO_PARENT(left, right, node.val)

dfs(root)
return self.ans
#4
BFS Level Order / Multi-Source Flood Fill
Guarantees shortest path; scales to grids, graphs, implicit state spaces
🔥 High Frequency Level-at-a-time Multi-source Implicit Graph

BFS's killer feature is level-order processing: you naturally know the distance from the source. Multi-source BFS (enqueue ALL sources at distance 0) solves "nearest X to every cell" in one pass.

🎯 Trigger

"Minimum steps/moves" · "Shortest path in unweighted graph" · "Spread from multiple sources" · "Level-by-level processing" · Grid problems

📌 Example — Word Ladder

Each word is a node. Two words are connected if they differ by one letter. BFS finds shortest transformation. The graph is implicit — you don't build it explicitly.

❌ Common Mistake

Marking visited on dequeue instead of enqueue. This lets duplicate nodes pile up in the queue, turning O(V+E) into O(V²) for dense graphs.

⏱ Saves

Rotting Oranges is 3 lines with multi-source BFS. Pacific Atlantic Water Flow runs two reverse-BFS from each coast — where they meet is the answer.

#5
Sliding Window + Frequency Map
Maintain a valid window by expanding right and shrinking left
🔥 High Frequency Fixed Window Variable Window At-most-k

Avoid recomputing from scratch by maintaining a running count of what's inside the window. Expand the right pointer always; shrink from the left only when the window violates a constraint.

🎯 Trigger

"Longest/shortest subarray/substring satisfying condition" · "Contains all characters of T" · "Minimum window covering" · "At most k distinct characters"

📌 Example — Minimum Window Substring

Track need (chars still required). Expand right until window is valid (need==0). Shrink left to minimize. Record min window at each valid state.

❌ Common Mistake

Shrinking the left pointer in a while loop when a single if would do (for "at most k"). Causes O(n²). Or updating the frequency map in the wrong direction.

⏱ Saves

Turns O(n²) substring search into O(n). Substring with Concatenation of All Words is sliding window with a word-level frequency map instead of char-level.

Python 2 — Variable Sliding Window Template
left = 0
window = {}   # or Counter

for right in xrange(len(s)):
    # 1. Add s[right] to window state
    window[s[right]] = window.get(s[right], 0) + 1

    # 2. Shrink from left while window is invalid
    while WINDOW_INVALID(window):
        window[s[left]] -= 1
        if window[s[left]] == 0:
            del window[s[left]]
        left += 1

    # 3. Window [left, right] is now valid — update answer
    ans = max(ans, right - left + 1)
TIER 2

High Value — Power Patterns

Appears in 6–10 problems, high signal
#6
Monotonic Stack
Maintain a stack that is always increasing (or decreasing) — pops trigger the answer
Next Greater Element Trapping Rain Water Heights / Boundaries

Keep a stack of indices in monotone order. When the current element breaks the monotonicity, pop and compute the answer for the popped element using the current element as its boundary.

🎯 Trigger

"For each element, find the next greater/smaller" · "Maximum rectangle in histogram" · "Trapping rain water" · Any problem where an element's answer depends on what comes next

📌 Example — Trapping Rain Water

Maintain a stack of decreasing heights. When height[i] > stack top, water is trapped between current element and the new stack top. Area = width × min_height.

❌ Common Mistake

Forgetting to handle elements left in the stack at the end (they have no right boundary). Add a sentinel 0 at the end, or handle the flush separately.

⏱ Saves

Turns O(n²) "for each element, scan forward" into O(n) one-pass. Also used in Simplify Path (directory stack), Valid Parentheses (matching stack).

#7
Backtracking: Choose → Explore → Unchoose
The universal template for exhaustive search with pruning
Permutations Subsets Combination Sum Generate Parentheses

All 7 backtracking problems use the same skeleton. The variation is only in: base case, the choices available, and the pruning condition. Get the skeleton correct once; reuse it everywhere.

🎯 Trigger

"Generate all..." · "Find all valid..." · "Count all combinations/permutations/partitions" — problems where you must enumerate all possibilities

📌 Example — Generate Parentheses

State = (current_str, open_count, close_count). Prune: don't add ( if open==n; don't add ) if close==open. No explicit "unchoose" since strings are immutable.

❌ Common Mistake

Forgetting the unchoose step (e.g., not popping from path after recursion). This corrupts sibling branches and produces wrong results silently.

⏱ Saves

Sorted input + skip-duplicate logic (if i > start and nums[i]==nums[i-1]: continue) handles all deduplication. Works for Subsets II, Combination Sum II, Permutations II.

Python 2 — Universal Backtracking Template
def backtrack(start, path, result):
    if BASE_CASE:
        result.append(list(path))   # deep copy!
        return
    for i in xrange(start, len(candidates)):
        if PRUNE_CONDITION: continue   # early exit
        path.append(candidates[i])    # choose
        backtrack(NEXT_START, path, result)
        path.pop()                    # unchoose

result = []
backtrack(0, [], result)
return result
#8
DP State Machine
Define discrete states; transitions depend on the current decision
House Robber Buy/Sell Stock Decode Ways Target Sum

Some DP problems have constraints on consecutive choices (can't rob adjacent houses; can't buy after selling same day). Model as a state machine: each state represents what you last did, and transitions are legal moves.

🎯 Trigger

"Can't take two in a row" · "Must wait k steps" · "Track whether item is currently held" · "Path with forbidden consecutive moves"

📌 Example — Stock with Cooldown

States: HELD, SOLD (cooldown), REST. Transitions: HELD→SOLD (sell), SOLD→REST, REST/SOLD→HELD (buy). dp[state] = max profit in that state.

❌ Common Mistake

Initializing states to 0 when they should be -inf (unreachable states). A state that's never valid should never be selected by max/min.

⏱ Saves

The house-robber pattern (dp[i] = max(dp[i-1], dp[i-2] + nums[i])) reduces to 2 variables, not an array. Space O(1) instead of O(n).

#9
Binary Search on Answer (not on Index)
When the feasibility of an answer is monotone, binary search the answer space
Koko Bananas Minimize Maximum Feasibility Check

Classic binary search finds a value in a sorted array. This variant asks: "what is the minimum k such that condition(k) is true?" If condition is monotone (false, false, ..., true, true, ...), binary search the answer.

🎯 Trigger

"Minimize the maximum" · "Maximize the minimum" · "What is the smallest k where X is possible?" · "Can we achieve X within budget k?"

📌 Example — Koko Eating Bananas

Binary search speed k from 1 to max(piles). For each k, check if Koko can finish all piles in h hours. Find the minimum valid k.

❌ Common Mistake

Off-by-one in the binary search bounds. Use lo, hi = MIN_POSSIBLE, MAX_POSSIBLE and converge to the first true with while lo < hi: mid = (lo+hi)//2.

⏱ Saves

Turns O(max_val × n) brute force into O(n log max_val). Also applies to Find Peak Element (binary search on unsorted but unimodal array).

#10
Prefix Sum + Running State
Precompute cumulative state; use subtraction for O(1) range queries
Subarray Sum = k Product Array Running Max/Min

Not just for sums — "running state" generalizes to prefix products, prefix XOR, prefix max. Combined with a hash map, it answers "how many subarrays have property P?" in one pass.

🎯 Trigger

"Subarray sum equals k" · "Product of array except self" · "Subarray divisible by k" · Any range aggregate on a static array

📌 Example — Product of Array Except Self

Left pass: left[i] = product of all nums[0..i-1]. Right pass: multiply by suffix product. No division needed, handles zeros.

❌ Common Mistake

Forgetting the sentinel prefix[0] = 0. Without it, subarrays starting at index 0 require special casing in every query.

⏱ Saves

Prefix sum + hash map turns "count subarrays with sum k" from O(n²) into O(n). The trick: count += seen[prefix - k] at each step.

TIER 3

Specialist Patterns

Appears in 3–6 problems, high specificity
#11
Sentinel / Dummy Head Node
Eliminates all "is head null?" special cases in linked list problems
🔥 Instant Bug Prevention Linked List Merge Remove Nth

Create a fake node dummy = ListNode(0); dummy.next = head. Now you never need to check if head is None before operating. Return dummy.next.

🎯 Trigger

Any linked list problem that modifies or returns the head: Merge Two Sorted Lists, Remove Nth Node, Partition List, Remove Duplicates II

⏱ Saves

Eliminates 4–6 lines of null-checking boilerplate. More importantly, prevents a whole class of "what if the answer is the new head?" bugs. Used in 8+ linked list problems.

#12
Fast / Slow Pointer (Floyd's)
Two pointers moving at different speeds detect cycles and find midpoints
Cycle Detection Find Middle Happy Number
🎯 Trigger

"Does this linked list have a cycle?" · "Find the middle node" · "Happy Number" (cycle in sequence) · Any problem with a repeated-state or cycle question

📌 Example — Linked List Cycle

slow = slow.next; fast = fast.next.next. If they meet, there's a cycle. If fast reaches None, no cycle. Never needs extra space.

❌ Common Mistake

Checking fast.next without checking fast first — causes AttributeError on NoneType. Always check: while fast and fast.next.

⏱ Saves

O(1) space vs O(n) visited set for cycle detection. Happy Number uses the same logic on digit-sum sequences.

#13
Topological Sort + In-degree BFS (Kahn's)
Process nodes with no dependencies first; detect cycles via leftover nodes
Course Schedule DAG ordering Cycle Detection
🎯 Trigger

"Given prerequisites, is this possible?" · "What order should we process these?" · Any dependency/scheduling problem in a directed graph

📌 Example — Course Schedule

Build in-degree array. Queue all nodes with in-degree 0. Process: for each neighbor, decrement in-degree; if 0, add to queue. If all nodes processed → no cycle.

⏱ Saves

Kahn's algorithm doubles as cycle detection (if processed count < n, there's a cycle). Covers both Course Schedule I and II with the same code.

#14
Two Heaps — Size-Balanced Partition
Split data into two halves; the boundary gives the median in O(log n)
Find Median from Data Stream Sliding Window Median
🎯 Trigger

"Running median" · "Find median after each insertion" · "Kth largest in a stream" · Any problem where you need fast access to both extremes of a dynamic dataset

📌 Example — Find Median from Data Stream

Max-heap for lower half, min-heap for upper half. Invariant: sizes differ by at most 1 and max(lower) ≤ min(upper). Rebalance after every insert.

⏱ Saves

Median in O(log n) per insert vs O(n) for sorted insertion or O(n log n) rebuild. The rebalancing logic is always the same — memorize it once.

#15
Interval Sort + Greedy Merge
Sort by start; merge when current interval overlaps the previous end
Merge Intervals Minimum Arrows Non-overlapping
🎯 Trigger

"Merge overlapping intervals" · "Minimum number of groups/arrows/meeting rooms" · "Insert interval into sorted list" · Any problem with ranges [start, end]

📌 Example — Minimum Arrows to Burst Balloons

Sort by end. Greedily shoot at the end of the current balloon. Any balloon whose start ≤ current arrow position is also burst. Count shots.

⏱ Saves

All four interval problems use sort-then-greedy. The specific condition varies (overlap vs containment vs touch) but the skeleton is identical.

#16
Union Find — Incremental Connectivity
Near O(1) dynamic component merging; detects cycles via same-root check
Redundant Connection Number of Components Graph Valid Tree
🎯 Trigger

"Are these two nodes connected?" · "How many connected components?" · "Does this edge create a cycle?" · Edges arrive one at a time (dynamic connectivity)

⏱ Saves

BFS/DFS rebuild connectivity from scratch — O(V+E) per query. Union Find maintains it incrementally — O(α) per edge. Covers Redundant Connection, Graph Valid Tree, Number of Connected Components.

#17
Greedy "Sort by Key, Then Decide"
Ordering often transforms a hard decision problem into an easy greedy one
Task Scheduler Partition Labels Gas Station Jump Game
🎯 Trigger

"Optimal scheduling" · "Minimum number of groups/rooms" · "Can we reach the end?" · "Assign tasks to minimize/maximize objective"

📌 Example — Jump Game II

Track farthest reachable and current end of level. When i reaches end, increment jumps, update end to farthest. One pass, O(n).

⏱ Saves

Gas Station: if total gas ≥ total cost, a solution always exists. Start from the first station after the deficit — greedy O(n). No DP needed.

TIER 4

Deep Cut Patterns

Specialized but high signal when triggered
#18
DFS State Modeling
The "state" in the DFS isn't always (node) — it's (node + extra context)

For many graph problems, the hard part is defining what a "node" is. In Word Ladder, a node is a word. In Pacific Atlantic, you run DFS backwards from each ocean. In Word Search, a node is (row, col, index in word). Always ask: "what uniquely defines a position in my search?"

Pacific Atlantic trickRun BFS/DFS from all Pacific borders and all Atlantic borders separately. Mark cells reachable from each ocean. Answer = intersection. Reverse-flow thinking saves you from the hard forward direction.
#19
Trie + DFS for Prefix Matching
Index a dictionary by prefix for O(L) lookup; prune DFS using the Trie

Word Search II would TLE with brute-force DFS over all words. Insert all words into a Trie; DFS the grid while simultaneously traversing the Trie. The moment the Trie has no child for the current character, stop — massive pruning.

Implementation shortcutUse a Python dict as the Trie node: node = {}. Terminal marker: node['#'] = word. This is faster to code than a class in interviews.
#20
Interval DP — "What Was Processed Last?"
Solve problems by asking which item was processed last in a range

For Burst Balloons →, instead of "which balloon bursts first?" (hard), ask "which balloon bursts last in range [l,r]?" (easy). The last balloon has neighbors l-1 and r+1 which are definitely still present — computable.

Pattern recognitionInterval DP applies when: the problem involves a range [l, r]; you make a choice within the range; the cost of that choice depends on what's left. Recurrence: dp[l][r] = max/min over k in [l..r] of: cost(k) + dp[l][k-1] + dp[k+1][r]. O(n³) time, O(n²) space.

→ Minimal Study Order by Pattern Reuse

Learn patterns that unlock the most problems first. This order maximizes problems you can solve after each session.

1
Hash Map + Two Pointers + Sliding Window
Unlocks: Two Sum, Group Anagrams, Valid Anagram, 3Sum, Container with Most Water, Longest Substring, Minimum Window. ~30 problems solved after week 1.
2
Tree Recursion Return Value + BFS Level Order
Unlocks all 18 Binary Tree problems and most Graph BFS problems (Rotting Oranges, Level Order, Right Side View). One template, ~25 problems.
3
Dummy Head + Fast/Slow Pointer (Linked Lists)
Unlocks all 11 linked list problems. Two patterns cover the entire category. Also: LRU Cache (hash map + doubly linked list).
4
Backtracking Template + DFS State Modeling
Unlocks all 7 backtracking problems + Graph DFS problems (Word Search, Clone Graph, Pacific Atlantic, Course Schedule with DFS). ~20 problems.
5
DP State Machine + Binary Search on Answer
Unlocks most of the 22 DP problems. Start with 1D DP (Climbing Stairs, Coin Change, House Robber), then 2D (LCS, Edit Distance). Binary Search on Answer covers Koko + rotated search.
6
Topo Sort + Union Find + Two Heaps + Interval Merge
Cleans up remaining graph problems (Course Schedule), all interval problems, Find Median from Data Stream, Redundant Connection. Hard hards: Burst Balloons (Interval DP), Word Search II (Trie + DFS).

⚡ Pre-Interview Pattern Checklist

  • Is this a "find pair / group / count" → Hash Map O(1)
  • Sorted array, find two elements → Two Pointers squeeze
  • Substring/subarray with a "window condition" → Sliding Window + freq map
  • Binary tree aggregation → post-order return value + global answer
  • Shortest path, unweighted → BFS · weighted → Dijkstra
  • Generate all combinations / permutations → backtracking template
  • Linked list modify/return head → dummy head first, always
  • Minimize maximum / maximize minimum → binary search on answer
  • DP on sequence with skip constraint → state machine (hold/rest/sold)
  • Prerequisites / ordering → topological sort (Kahn's in-degree)
