Majority Element LC #169

⏱ 5 min🎯 Boyer-Moore Voting📌 Frequency: High

1 Problem

ProblemGiven an array of size n, find the element that appears more than n/2 times. It is guaranteed to exist.

Example: [3,2,3] → 3  |  [2,2,1,1,1,2,2] → 2

2 Key Insight — Boyer-Moore Voting

The trickImagine an election where every voter can "cancel out" an opposing voter. The majority element has more votes than all others combined — it survives the cancellation. Keep a candidate and a count. When count hits 0, switch to the current element. When current equals candidate, increment; otherwise decrement.

The key invariant: the majority element's net positive votes always exceed the total of all cancellations.

🗳️

Why It Works

The majority element appears >n/2 times. Even if it's paired against every other element, it still has votes left over at the end.

⚡

O(1) Space!

No hash map needed. Just two variables: candidate and count. This is the O(n) time, O(1) space solution.

4 Python 2 Code

Python 2 — Boyer-Moore O(n) time O(1) space
class Solution(object):
    def majorityElement(self, nums):
        candidate = None
        count = 0

        for num in nums:
            if count == 0:
                candidate = num     # reset candidate
            if num == candidate:
                count += 1          # same team
            else:
                count -= 1          # cancel out

        return candidate   # guaranteed majority exists

Hash Map (simpler, O(n) space)

Python 2 — O(n) time, O(n) space
from collections import Counter

class Solution(object):
    def majorityElement(self, nums):
        counts = Counter(nums)
        return max(counts, key=counts.get)
        # or: return sorted(nums)[len(nums)//2]  — O(n log n)

⚡ 1-Minute Review

  • Boyer-Moore: candidate + count — only 2 variables
  • count==0 → reset candidate to current element
  • same as candidate → count++; else → count--
  • The majority element always survives cancellations (it has > n/2 votes)
  • O(n) time, O(1) space
