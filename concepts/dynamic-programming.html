Dynamic Programming

⏱ 10 min revision🎯 Overlapping Subproblems📌 Frequency: Very High

1 Concept Overview

Dynamic Programming (DP) solves complex problems by breaking them into overlapping subproblems and caching their results. Instead of recomputing the same subproblem multiple times (like naive recursion), DP computes each subproblem exactly once.

Two approaches: Top-down (memoization) — recursion + cache; Bottom-up (tabulation) — fill a table iteratively from base cases up.

Two required properties for DP1. Overlapping subproblems — same subproblems recur (not like Divide & Conquer where subproblems are independent). 2. Optimal substructure — optimal solution of the problem contains optimal solutions of subproblems.

2 Why Interviewers Ask It

DP problems constitute 30–40% of LeetCode Hard problems. They test whether you can identify the state space, define a recurrence relation, and handle base cases. It's the deepest algorithmic skill tested at senior levels — correctly identifying the DP state separates strong candidates from the rest.

3 Pattern Recognition

Use DP when you see"Count the number of ways" · "Minimum/maximum of something" · "Can you achieve X?" · Problems with choices at each step · "Is this possible?" (boolean DP) · String matching, knapsack, coin change, path problems

Ask yourself: "If I knew the answer to a smaller version of this problem, could I build the answer to the full problem?" If yes, that's DP.

Red flag for backtracking instead: "find ALL solutions" or "list every combination" — DP counts/optimizes, backtracking enumerates.

4 Core Idea

The DP problem-solving framework:

1

Define the state

dp[i] = ? What does this index represent? Be precise. "dp[i] = minimum cost to reach index i" is a valid state. "dp[i] = something with i" is not.

2

Write the recurrence

How do we compute dp[i] from smaller subproblems? dp[i] = min(dp[i-1] + cost[i], dp[i-2] + cost[i])

3

Identify base cases

What are the smallest valid values? dp[0] = 0, dp[1] = cost[0]

4

Determine traversal order

Bottom-up: ensure subproblems are solved before the current problem. Usually left-to-right, or outer-to-inner for 2D.

7 Common Tricks

🔄

Space Optimization

If dp[i] only depends on dp[i-1] and dp[i-2], you only need 2 variables, not an array. Common in Fibonacci, Climbing Stairs, House Robber.

↔️

Rolling Array for 2D

If dp[i][j] only depends on dp[i-1][j], you can use two 1D arrays (current and previous row), reducing space from O(mn) to O(n).

📝

Top-down First

Write the recursive solution first, add @lru_cache or a memo dict — done. Then convert to bottom-up if needed for constant space or interview style.

🔢

State Definition is Everything

The hardest part is the DP state. Ask: "what information do I need at step i to make the optimal decision?" Those variables are your state.

8 Common Mistakes

Mistake 1 — Wrong recurrence directionFor coin change, you go from amount 1 to target. Going backwards won't work. Always think about which direction ensures subproblems are solved before you need them.
Mistake 2 — Incorrect base casesdp[0] is often 0 (cost) or 1 (count). Getting base cases wrong causes wrong answers that pass most test cases. Think carefully: what does dp[0] mean?
Mistake 3 — Off-by-one in DP table sizedp of size n vs n+1. For "amount" problems, dp needs size amount+1 to have dp[amount] available.
Mistake 4 — Greedy when DP is requiredGreedy works when local optimum = global optimum. Greedy fails for problems like coin change with arbitrary denominations. When in doubt, think DP.

9 Python 2 Template

Top-down (Memoization)

Python 2
# Coin Change — Top-down with memo
def coinChange(coins, amount):
    memo = {}

    def dp(remaining):
        if remaining == 0: return 0
        if remaining < 0:  return float('inf')
        if remaining in memo: return memo[remaining]

        best = float('inf')
        for coin in coins:
            res = dp(remaining - coin)
            if res != float('inf'):
                best = min(best, res + 1)

        memo[remaining] = best
        return best

    result = dp(amount)
    return result if result != float('inf') else -1

Bottom-up (Tabulation)

Python 2
# Coin Change — Bottom-up tabulation
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0   # base case: 0 coins to make amount 0

    for amt in xrange(1, amount + 1):
        for coin in coins:
            if coin <= amt and dp[amt - coin] != float('inf'):
                dp[amt] = min(dp[amt], dp[amt - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1

Longest Common Subsequence (2D DP)

Python 2
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in xrange(m + 1)]

    for i in xrange(1, m + 1):
        for j in xrange(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1      # characters match
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])   # skip one

    return dp[m][n]

0/1 Knapsack

Python 2
def knapsack(weights, values, capacity):
    n = len(weights)
    # dp[i][w] = max value using first i items with capacity w
    dp = [[0] * (capacity + 1) for _ in xrange(n + 1)]

    for i in xrange(1, n + 1):
        for w in xrange(capacity + 1):
            # don't take item i
            dp[i][w] = dp[i-1][w]
            # take item i (if it fits)
            if weights[i-1] <= w:
                dp[i][w] = max(dp[i][w], dp[i-1][w - weights[i-1]] + values[i-1])

    return dp[n][capacity]

10 Python 2 Variations

House Robber (Space-optimized)

Python 2
def rob(nums):
    prev2 = prev1 = 0
    for n in nums:
        curr  = max(prev1, prev2 + n)
        prev2 = prev1
        prev1 = curr
    return prev1

Longest Increasing Subsequence (O(n log n))

Python 2
import bisect

def lis(nums):
    tails = []
    for n in nums:
        pos = bisect.bisect_left(tails, n)
        if pos == len(tails):
            tails.append(n)
        else:
            tails[pos] = n
    return len(tails)

12 Step-by-Step Walkthrough

Problem: Coin change — coins=[1,2,5], amount=5. Min coins?

1

Initialize

dp = [0, ∞, ∞, ∞, ∞, ∞]. dp[0]=0 (base case: 0 coins to make amount 0).

2

Fill dp[1]

Try coin=1: dp[1-1]+1 = dp[0]+1 = 1. dp[1] = 1.

3

Fill dp[2]

coin=1: dp[1]+1=2. coin=2: dp[0]+1=1. dp[2] = min(2,1) = 1.

4

Fill dp[3], dp[4]

dp[3]=2 (coin=1+dp[2] or coin=2+dp[1]). dp[4]=2 (coin=2+dp[2]).

5

Fill dp[5]

coin=5: dp[0]+1=1. dp[5] = 1. Answer: 1 coin (just the 5-coin).

13 Mental Model

Think of DP as "lazy evaluation with a notebook"Every time you solve a subproblem, you write the answer in a notebook. Next time someone asks for the same subproblem, you just look it up. The insight is that the optimal answer to the big problem is always built from the optimal answers to smaller problems — you never need to reconsider a solved subproblem.

14 Revision Notes

⚡ 1-Minute Review

  • Two properties: overlapping subproblems + optimal substructure
  • Define state precisely: dp[i] = "exact meaning"
  • Write recurrence: how does dp[i] depend on smaller dp values?
  • Top-down = recursion + memo dict (easier to code)
  • Bottom-up = loop filling table from base cases (better space)
  • Space optimization: if dp[i] only uses dp[i-1], use 2 variables
  • Common patterns: 1D linear, 2D grid, knapsack, interval, string
  • "Count ways" → usually dp[i] = sum. "Min/max" → dp[i] = min/max.
