Two Sum LC #1

⏱ 5 min revision🎯 Hash Map Complement📌 Frequency: Extremely High

1 Problem Summary

ProblemGiven an array of integers nums and an integer target, return the indices of the two numbers that add up to target. Exactly one solution exists, and you cannot use the same element twice.

Example: nums = [2, 7, 11, 15], target = 9 → [0, 1] (because nums[0] + nums[1] = 2 + 7 = 9)

2 Pattern

Hash Map ComplementFor each number, you need its complement (target - num). Store numbers you've already seen in a hash map (value → index). For each new number, check if its complement is already in the map.

This pattern generalizes to: "Find two elements satisfying a condition" where you can precompute what you're looking for in O(1) per element.

3 Visualization

4 Brute Force — O(n²)

Check every pair. Simple but too slow for large inputs.

Python 2
class Solution(object):
    def twoSum(self, nums, target):
        n = len(nums)
        for i in xrange(n):
            for j in xrange(i + 1, n):
                if nums[i] + nums[j] == target:
                    return [i, j]
Why this failsO(n²) time — for n=100,000 that's 10^10 operations. Always mention this to the interviewer and say you can do better.

5 Optimal Solution — O(n)

One pass with a hash map. For each element, check if its complement (target - num) is already stored. If yes, return the pair.

Key insightInstead of looking forward ("is there a j > i such that nums[j] = target - nums[i]?"), we look backward ("have I already seen target - nums[i]?"). Same answer, but O(1) per lookup.

6 Python 2 Code

Python 2 — Optimal O(n)
class Solution(object):
    def twoSum(self, nums, target):
        seen = {}   # value → index

        for i, num in enumerate(nums):
            complement = target - num
            if complement in seen:
                return [seen[complement], i]
            seen[num] = i

        return []   # guaranteed to have a solution per problem statement

7 Dry Run

Input: nums=[2,7,11,15], target=9

1

i=0, num=2

complement = 9-2 = 7. seen={} → 7 not found. Add seen={2:0}.

2

i=1, num=7

complement = 9-7 = 2. seen={2:0} → 2 found at index 0! Return [0, 1]. ✓

8 Complexity Analysis

SolutionTimeSpace
Brute force (two loops)O(n²)O(1)
Sort + two pointers (if indices not needed)O(n log n)O(1)
Hash map (optimal, indices needed)O(n)O(n)
Interview noteIf the problem asked for values (not indices), sort + two pointers would be O(n log n) time and O(1) space. Since it asks for indices, you can't sort — hash map is the way.

9 Variations

🔢

Two Sum II (sorted array)

Use two pointers. Move left forward if sum too small, right backward if too large. O(n) time, O(1) space.

🔄

Three Sum

Fix one element, then use two pointers on the rest. O(n²) time. Watch for duplicates with pointer skipping.

🌐

Two Sum in BST

In-order traversal to get sorted array, then two pointers. Or use a hash set and BST in-order DFS.

✅

Two Sum (count pairs, not indices)

Use Counter. For each num, count how many times complement appears. Avoid double-counting by dividing by 2 for equal pairs.

Two Pointers variant (sorted array, values only)

Python 2
class Solution(object):
    def twoSum(self, numbers, target):   # LC 167 — sorted array
        left, right = 0, len(numbers) - 1
        while left < right:
            s = numbers[left] + numbers[right]
            if s == target:
                return [left + 1, right + 1]   # 1-indexed
            elif s < target:
                left  += 1
            else:
                right -= 1
